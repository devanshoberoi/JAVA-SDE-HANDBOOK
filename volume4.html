Chapter 2: Single Responsibility Principle (SRP)

The Foundation Of Clean Architecture, Maintainable Software, High Cohesion, and Professional Object-Oriented Design.

1. What Is SRP?
A class should have: 

ONE Responsibility

ONE Reason To Change
Nothing more. Nothing less.

Official Definition


A class should have
only one reason to change.
Created by: 
Robert C. Martin
(Uncle Bob)
2. Why SRP Exists
Imagine a restaurant employee. Bad design: 

Cook Food

Take Orders

Clean Tables

Manage Payroll

Handle Marketing
One person doing everything. Disaster. Same thing happens in software.
3. Real World Analogy
Hospital Example: 

Doctor

↓

Treat Patients
Accountant: 

Manage Finances
Receptionist: 

Handle Appointments
Each role has one responsibility. That is SRP.
4. Bad Design Example

   createUser();

   saveToDatabase();

   sendEmail();

   generateReport();
}

Reporting Logic

↓

Software should be: 

OPEN

For Extension

For Modification

2. Why OCP Exists
Imagine:

↓

↓

3. Real World Analogy

Tablet

Power Bank

Chapter 4: Liskov Substitution Principle (LSP)

Correct Inheritance Design, Behavioral Compatibility, Composition vs Inheritance, and Avoiding Broken Hierarchies.

1. What Is LSP?
If class B extends class A, then class B should be usable everywhere class A is expected. Without breaking the program.

Official Definition


Objects of a subclass

should replace

objects of the superclass

without affecting correctness.
2. Why LSP Exists
Inheritance often looks correct. But behavior may become incorrect. Example: 

Animal

↓

Bird

↓

Penguin
Looks reasonable. But: 

Bird Can Fly

Penguin Cannot Fly
Problem.
3. Real World Analogy
Imagine: 

Employee

↓

Manager
If the company software expects: 

Any Employee

Can Work
then every Manager should also be able to work correctly. Otherwise inheritance is broken.
4. Simple Valid Example 
class Animal
{
   void eat()
   {
   }
}

class Dog
extends Animal
{
}
Anywhere: 
Animal
is expected, 
Dog
works correctly. LSP satisfied.
5. First Broken Example

{

   {
   }
}

class Penguin
extends Bird
{
}

Penguins

Cannot Fly

Suppose:

Bird bird)

   bird.fly();
}
Now:

new Penguin();

penguin
);
Behavior is incorrect. LSP violated.
7. The Famous Rectangle-Square Problem
Most famous LSP interview question. Many people design:

↓
